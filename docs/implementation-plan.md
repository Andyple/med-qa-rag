## Phase 2 — Ingestion pipeline
- [ ] Implement NCBI E-utilities client (fetch abstracts by keyword or MeSH term)
- [ ] Implement chunking: paragraph-first, sentence-boundary fallback at 512 tokens
- [ ] Implement embedding wrapper (supports both text-embedding-3-small and PubMedBERT)
- [ ] Implement Chroma vector store adapter
- [ ] Write ingestion runner script with --limit flag
- [ ] Unit tests for chunker (edge cases: very short, very long, no paragraph breaks)
- [ ] Integration test: ingest 100 abstracts, verify chunk count and metadata

## Phase 3 — Retrieval
- [ ] Implement retrieval function: query → top-k chunks with similarity scores
- [ ] Return similarity scores alongside chunks (needed for uncertainty signal)
- [ ] Unit test: mock vector store, verify score passthrough
- [ ] Integration test: retrieve on a known query, inspect top results

## Phase 4 — Conflict detection
- [ ] Load NLI model (cross-encoder/nli-deberta-v3-small) with CPU fallback
- [ ] Implement pairwise contradiction scoring for retrieved chunks
- [ ] Set has_contradictions flag when any pair exceeds threshold
- [ ] Unit tests with synthetic agreeing and contradicting chunk pairs

## Phase 5 — Generation with self-consistency
- [ ] Write system prompt and user prompt templates in prompts.py
- [ ] Implement single-call generation with structured JSON output
- [ ] Implement N-sample self-consistency: run N times, compute agreement rate
- [ ] Define agreement metric (exact match is too strict; use semantic similarity)
- [ ] Unit test: mock LLM, verify agreement rate calculation

## Phase 6 — Claim-level attribution
- [ ] Integrate spaCy sentencizer for claim splitting
- [ ] Implement NLI scoring: (claim, chunk) → entailment score
- [ ] Implement attribution: for each claim, find best chunk and score
- [ ] Apply threshold, set UNSUPPORTED flag on low-scoring claims
- [ ] Unit tests: supported claim, unsupported claim, numerical claim edge case
- [ ] Integration test: run attribution on a known answer + chunks

## Phase 7 — Uncertainty aggregation
- [ ] Implement retrieval confidence signal (mean similarity)
- [ ] Implement generation confidence signal (agreement rate)
- [ ] Implement attribution confidence signal (fraction supported)
- [ ] Implement weighted aggregation; weights in config.py
- [ ] Unit test: verify each signal independently, then combined

## Phase 8 — FastAPI layer
- [ ] Define request/response schemas in schemas.py (use Pydantic)
- [ ] Implement POST /ask endpoint
- [ ] Implement GET /health endpoint
- [ ] Add request validation and error handling
- [ ] Integration test: start server, hit /ask with a real query

- [ ] Output eval_results.md with all metrics
